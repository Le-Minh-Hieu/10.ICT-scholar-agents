Moving to a vision-first architecture is **COMPONENT REORDERING**, not an architecture rewrite.

**95-99% of existing code can be reused**

### Required Work

1. **New Code (~230 lines):**
   - Vision formation function

   - KnowledgeGap type

2. **Modified Code (~150-200 lines):**
   - buildQueries() input source

3. **Unchanged Code (~49,500 lines):**
   - Entire retrieval pipeline
   - Entire ontology layer
   - All agents (minor prompt tweaks only)
   - All grounding logic
   - All utility functions

```
PHASE 1: Add Types
```
